# 01. Tokenization


## 한 줄 설명

Tokenization은 LLM이 이해할 수 있게 글자를 작은 조각으로 나누고, 각 조각에 번호를 붙이는 과정


## Token

Token은 모델이 다루는 가장 작은 텍스트 단위
    - 단어 전체일 수도 있고
    - 단어 일부일 수도 있고
    - 공백•기호도 하나의 토큰일 수 있다


## 예시 (개념만)

| 원문          | 토큰으로 나눈 모습 (예시)             |
|:-------------|:--------------------------------|
| Hello        | Hello                           |
| Hello world  | Hello + world                   |
| unhappiness  | un + happi + ness               |
| 고양이        | 모델마다 다름 (한 글자 또는 여러 조각)   |


중요: token은 "의미 있는 단어"와 1:1이 아님
모델이 자주 쓰는 패턴 기준으로 잘게 쪼갠 조각


## 비유

### 택배 상자 비유

- 문장 = 큰 택배 박스
- 토큰 = 박스 안 작은 포장 단위
- 토큰 ID(번호) = 창고 선반 번호

창고(LLM)는 "고양이"라는 글자가 아니라 "4521번 선반에 있는 물건"만 알아봄
Tokenization은 박스를 포장 단위로 나누고 선반 번호를 붙이는 작업

*   토큰 ID 하나 = 창고에 정해진 포장 1종류(항상 같은 글자 조각)
    문장 하나 = 그런 선반 번호를 여러 개, 순서대로 적은 리스트


## 나누는 방식 (BPE - 대표 방식)

Step 1: 글자 단위로 시작
```text
lower -> l, o, w, e, r
```

Step 2: 자주 붙어 나오는 조합을 하나로 합침
```text
"lo" 가 자주 나온다 -> "lo" 를 하나의 토큰으로 등록
"low" 도 자주 나온다 -> "low" 도 토큰으로 등록
```

Step 3: 반복
    - 자주 등장하는 짝(쌍)을 계속 묶어서
    - 사전을 수만 ~ 수십만 개 토큰으로 만듦

## 토큰화 방식 비교

| 방식           | 장점                                   | 단점                               |
|:---------------|:---------------------------------------|:-----------------------------------|
| 글자 하나씩    | 사전이 작음                            | 문장이 너무 길어짐                 |
| 단어 하나씩    | 직관적                                 | 새 단어·오타 처리 어려움           |
| BPE (중간)     | 적당한 길이 + 새 단어도 조각으로 처리  | 사람 눈엔 조각이 어색할 수 있음    |


## Tokenization -> Embedding으로 이어짐

```text
"고양이"  →  토큰 번호 4521  →  (다음 단계) 임베딩: 숫자 배열로 의미 표현
```

```text
"고양이"  →  [토큰 조각들]  →  [토큰 ID들]  →  [임베딩 벡터들]  →  Transformer ...
   글자        Tokenization      번호 목록        Embedding
```

> ** Tokenization 없이는 Embedding도, LLM 연산도 시작할 수 없습니다. 


## 실생활에서 느껴지는 Tokenization

### 같은 문장도 토큰 수가 다를 수 있음
    - 영어: "Hello World" -> 토큰 2~3개
    - 한글: 같은 의미라도 토큰이 더 많을 수 있음(모델·학습 데이터에 따라)
-> API 요금·속도가 "글자 수"가 아니라 "토큰 수" 기준인 이유

### 띄어쓰기·특수문자도 영향
```text
"안녕하세요" vs "안녕 하세요"
```
토큰 개수·조각이 달라질 수 있음

### 모델마다 토큰화 방식이 다름
    - GPT용 tokenizer ≠ Llama용 tokenizer
        -> 같은 문장도 모델마다 토큰 수가 다름




## 내가 이해한 내용

LLM은 글자를 직접 이해할 수 없다.

따라서 먼저 텍스트를 token(= 모델이 다루는 가장 작은 텍스트 단위)으로 나누고,
각 토큰을 고유한 번호 (Token ID)로 변환한다. 

여기서 토큰은 단어 전체가 될 수 있고, 단어 일부일 수 있고, 띄어쓰기나 기호일 수 있다.

고유한 번호(Token ID)들이 다음 단계인 Embedding의 입력이 된다.

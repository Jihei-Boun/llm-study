# 08. AI Agent

<br>

## 한 줄 설명

AI Agent = AI가 **한 번에 답만 하는 게 아니라**, 목표를 정하고 **스스로 계획을 세우고**, **도구를 써서** 일을 **여러 단계**로 처리하는 방식

<br>

## 이전 노트와의 연결

06번 **Inference & Generation**까지 배웠다면:

```text
질문 → AI가 답 한 번 생성 → 끝
```

07번 **RAG**는 그 앞에 **자료 검색**이 붙음:

```text
질문 → 자료 찾기 → AI가 답 생성 → 끝
```

**AI Agent**는 여기서 한 단계 더 나아감:

```text
목표 → (생각 → 행동 → 결과 확인) 반복 → 목표 달성
         ↑__________________________|
              필요하면 여러 번 반복
```

비유:

| 단계 | 06번 (일반 AI) | 07번 (RAG) | 08번 (Agent) |
|------|----------------|------------|--------------|
| 시험 | **암기한 것만**으로 답 | **교과서를 펼쳐** 답 | **문제를 풀기 위해** 계산기·노트·친구에게 전화까지 함 |
| 요리 | 레시피 **외워서** 요리 | **냉장고를 열어** 재료 확인 | **재료 확인 → 부족하면 장보기 → 요리 → 맛 보고 간 맞추기** |
| 일상 | "오늘 날씨 어때?"에 **한마디** | 회사 규정 **찾아서** 답 | "이번 주 회의 준비해줘" → **일정 확인 → 자료 검색 → 초안 작성 → 보내기** |

<br>

✅ 정리

> Inference & Generation이 **"한 번 말하기"**라면,  
> RAG는 **"찾아 읽고 한 번 말하기"**,  
> AI Agent는 **"목표를 위해 여러 번 행동하기"**

<br>

## AI Agent란?

**Agent(에이전트)** = "대리인", "일을 대신 처리하는 존재"

**AI Agent**는 큰 언어 모델(LLM)에 **뇌** 역할을 맡기고,  
**손과 발**(도구)과 **기억**, **판단력**을 더 붙인 것.

```text
┌─────────────────────────────────────────┐
│              AI Agent                    │
│                                         │
│   🧠 LLM (생각)                          │
│      "다음에 뭘 해야 하지?"               │
│                                         │
│   🔧 Tools (행동)                        │
│      검색, 계산, 파일 읽기, API 호출...   │
│                                         │
│   📋 Memory (기억)                       │
│      지금까지 한 일, 중간 결과             │
│                                         │
│   🎯 Goal (목표)                         │
│      "사용자가 원하는 최종 결과"           │
└─────────────────────────────────────────┘
```

파이썬을 몰라도 이해하면 됨:  
Agent는 **프로그래밍 언어**가 아니라 **AI가 일하는 방식(패턴)**의 이름.

<br>

## 일반 챗봇 vs RAG vs Agent

| | 일반 챗봇 | RAG | AI Agent |
|---|-----------|-----|----------|
| **입력** | 질문 한 번 | 질문 한 번 | 목표·작업 (복잡할 수 있음) |
| **하는 일** | 답만 생성 | 자료 찾고 답 생성 | 계획 → 여러 행동 → 결과 확인 |
| **단계 수** | 1단계 | 2단계 (검색 + 생성) | **여러 단계** (루프) |
| **외부 연결** | 없음 | 문서 검색 | 검색, 계산, 코드 실행, 이메일 등 **다양한 도구** |
| **비유** | 전화 한 통 | 도서관에서 책 찾아 답 | **비서**가 일을 끝까지 처리 |

<br>

## 핵심 동작 — "생각 → 행동 → 관찰" 루프

Agent의 심장은 **반복 루프**임.

```text
사용자: "서울 내일 날씨 알려주고, 비 오면 우산 챙기라고 알림 설정해줘"

[1] 생각 (Reasoning)
    "날씨를 먼저 알아야겠다 → 날씨 API를 써야겠다"

[2] 행동 (Action)
    → 날씨 도구 실행: "서울, 내일 → 비 예보"

[3] 관찰 (Observation)
    "결과: 비 80% 확률"

[4] 다시 생각
    "비가 온다 → 알림 설정 도구를 써야겠다"

[5] 다시 행동
    → 알림 도구 실행

[6] 관찰
    "알림 설정 완료"

[7] 최종 답변
    "내일 서울은 비 예보(80%)입니다. 우산 챙기라는 알림을 설정했습니다."
```

이 패턴을 **ReAct**(Reason + Act)라고 부르기도 함.  
이름보다 **"생각하고, 하고, 결과 보고, 또 생각한다"**만 기억하면 됨.

<br>

## 한 장면으로 보기

```text
  사용자 목표
  "이번 분기 매출 요약 보고서 만들어줘"
       │
       ▼
  ┌─────────────┐
  │  🧠 LLM     │  "먼저 매출 데이터가 어디 있는지 찾아야 해"
  └──────┬──────┘
         │ 행동 결정
         ▼
  ┌─────────────┐
  │  🔧 도구     │  파일 검색 → sales_Q1.csv 발견
  └──────┬──────┘
         │ 결과 관찰
         ▼
  ┌─────────────┐
  │  🧠 LLM     │  "표를 읽고 합계를 계산해야 해"
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │  🔧 도구     │  계산기 / 표 분석
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │  🧠 LLM     │  "요약 문장을 작성하고 파일로 저장하자"
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │  🔧 도구     │  문서 작성 + 저장
  └──────┬──────┘
         ▼
  최종 결과 → 사용자에게 전달
```

06번의 **토큰 생성**은 Agent 안에서도 계속 일어남.  
차이는 AI가 **"다음 글자"**만 고르는 게 아니라 **"다음에 할 일"**도 고른다는 것.

<br>

## Agent의 핵심 구성 요소

| 구성 요소 | 역할 | 일상 비유 |
|-----------|------|-----------|
| **LLM (뇌)** | 계획, 판단, 글 작성 | **두뇌** — "다음에 뭘 할까?" |
| **Tools (도구)** | 실제 세계와 상호작용 | **손·발** — 검색, 계산, 파일, API |
| **Memory (기억)** | 이전 대화·중간 결과 유지 | **메모장** — 방금까지 한 일 기억 |
| **Planning (계획)** | 큰 목표를 작은 단계로 쪼개기 | **할 일 목록** |
| **Goal (목표)** | 무엇을 달성할지 | **미션** |

<br>

## Tools(도구) — Agent에게 "손"을 주는 것

LLM 혼자는 **인터넷에 접속**, **파일 열기**, **정확한 계산**을 직접 못 함.  
(텍스트만 잘 만드는 **뇌**에 가깝기 때문)

그래서 Agent는 **도구 목록**을 AI에게 알려줌:

```text
[사용 가능한 도구]
1. web_search   — 웹에서 검색
2. calculator   — 숫자 계산
3. read_file    — 파일 내용 읽기
4. send_email   — 이메일 보내기
5. run_code     — 코드 실행 (고급)
```

AI는 매 단계마다 대략 이런 **선택**을 함:

```text
"지금은 web_search를 쓰겠다. 검색어는 '서울 내일 날씨'"
```

실제 검색·계산·파일 읽기는 **도구(외부 프로그램)**가 실행하고,  
결과만 AI에게 **다시 전달**됨.

**비유:** AI는 **지휘자**, 도구는 **악기 연주자**.  
지휘자가 "지금 바이올린!"이라고 하면 바이올린이 연주함.

<br>

## Memory(기억) — 두 종류

| 종류 | 내용 | 비유 |
|------|------|------|
| **단기 기억** | 지금 대화, 방금 도구 결과 | **책상 위 메모** |
| **장기 기억** | 사용자 취향, 과거 대화 요약 | **서랍 속 파일** |

Agent가 "아까 검색한 결과"를 기억해야 다음 단계로 넘어갈 수 있음.  
RAG의 **검색 결과를 프롬프트에 넣는 것**도 일종의 기억이지만,  
Agent는 **여러 번의 행동 결과**가 쌓임.

<br>

## 실생활 예시

### 1. Cursor (지금 쓰는 이 도구)

```text
목표: "버그 고쳐줘"

Agent 동작:
  → 코드베이스 검색
  → 관련 파일 읽기
  → 원인 추론
  → 코드 수정
  → 테스트 실행
  → 결과 확인 후 사용자에게 보고
```

### 2. 여행 계획 비서

```text
목표: "다음 주 제주도 2박 3일 일정 짜줘"

  → 날씨 검색
  → 항공·숙소 정보 검색
  → 일정표 작성
  → 예산 계산
  → 최종 일정 제안
```

### 3. 고객 지원 Agent

```text
목표: "환불 요청 처리"

  → 주문 번호 확인
  → 주문 DB 조회
  → 환불 정책 문서 검색 (RAG)
  → 환불 가능 여부 판단
  → 환불 API 호출
  → 고객에게 결과 안내
```

## RAG와 Agent — 같기도 하고 다르기도

RAG는 Agent **안에 들어갈 수 있음**.

```text
Agent
  ├── 도구 1: web_search
  ├── 도구 2: calculator
  └── 도구 3: rag_search  ← 07번 RAG가 "도구" 하나로 붙은 것
```

| | RAG | Agent |
|---|-----|-------|
| **핵심** | 자료 **검색 후** 답 | **목표 달성**까지 여러 행동 |
| **흐름** | 검색 → 생성 (1~2단계) | 생각 → 행동 → 관찰 **반복** |
| **관계** | Agent의 **도구 중 하나**일 수 있음 | RAG보다 **더 큰 그림** |

<br>

## Multi-Agent — Agent 여러 명이 협력

복잡한 일은 **역할 나누기**도 함.

```text
사용자: "앱 새 기능 기획하고 코드까지 작성해줘"

  📋 기획 Agent  → 요구사항 정리
  🎨 디자인 Agent → 화면 구조 제안
  💻 개발 Agent  → 코드 작성
  🔍 검토 Agent  → 버그·품질 점검
```

**비유:** 회사에서 **팀 프로젝트** — 한 사람(Agent)이 전부 하기 어려우면 나눔.

<br>

## Agent가 특히 잘 맞는 경우

- **여러 단계**가 필요한 작업 (조사 → 정리 → 실행)
- **최신 정보**가 필요한 작업 (검색 도구 사용)
- **정확한 계산·데이터 처리** (계산기, DB 도구)
- **반복·지루한 업무** 자동화 (파일 정리, 보고서 초안)

<br>

## Agent의 한계와 주의점

### 1. 잘못된 계획

AI가 **엉뚱한 순서**로 일하거나 **불필요한 단계**를 반복할 수 있음.

### 2. 도구 오용

"검색" 대신 "추측"으로 답하거나, **잘못된 도구**를 고를 수 있음.

### 3. 무한 루프

목표에 도달 못하고 **같은 행동을 계속**할 수 있음.  
→ 보통 **최대 반복 횟수** 제한을 둠.

### 4. 비용·시간

한 번의 질문에 **여러 번** LLM 호출 + 도구 실행 → **느리고 비쌀 수** 있음.

### 5. 보안

파일 읽기, 코드 실행, API 호출 권한 → **신뢰할 수 있는 환경**에서만 사용.

<br>

✅ 정리

> Agent는 **만능 비서**처럼 보이지만,  
> **감독(사람)**이 목표를 명확히 하고, 결과를 **확인**하는 게 중요함.

<br>

## 전체 그림 — 01~08번 한눈에

```text
01 Tokenization     글자 → 토큰
02 Embedding        의미 → 숫자
03 Attention        중요한 부분에 집중
04 Transformer      위를 묶은 AI 뼈대
05 Next Token       다음 글자 고르기
06 Inference        질문 → 답 (한 번)
07 RAG              찾아 읽고 → 답
08 AI Agent         생각 → 행동 → 반복 → 목표 달성
```

<br>

## 핵심만 다시

1. **AI Agent** = LLM + **도구** + **기억** + **반복 루프**
2. **한 번에 답**이 아니라 **목표까지** 여러 단계로 처리
3. **ReAct** = 생각(Reason) → 행동(Act) → 결과 관찰 → 반복
4. **RAG**는 Agent가 쓸 수 있는 **도구 중 하나**
5. **Cursor** 같은 코딩 AI도 Agent의 대표적인 예
6. 파이썬 몰라도 됨 — **개념**은 "스스로 계획하고 도구 쓰는 AI"

<br>

## 더 알아볼 키워드 (선택)

| 키워드 | 한 줄 설명 |
|--------|------------|
| **Function Calling** | AI가 "이 함수 실행해줘"라고 **구조화된 요청**을 보내는 방식 |
| **ReAct** | Reasoning + Acting — 생각과 행동을 번갈아 하는 패턴 |
| **Plan-and-Execute** | 먼저 **전체 계획** 세우고, 단계별 실행 |
| **Tool Use** | LLM이 외부 도구를 **선택·호출**하는 능력 |
| **Human-in-the-loop** | 중요한 단계에서 **사람이 승인** 후 진행 |

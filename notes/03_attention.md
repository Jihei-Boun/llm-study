# 03. Attention

<br>

## 한 줄 설명

Attention = 문장을 읽을 때, 지금 단어가 다른 단어들을 얼마나 신경 쓸 지 정하는 것

<br>

## 왜 필요한가?

### 사람

> 은행에서 돈을 찾았다.

***돈***을 이해하려면 ***은행***을 같이 봐야함.  
"돈"만 따로 보면 의미가 애매

> 강가에서 은행을 걸었다. 

***은행***을 이해하려면 ***강가***를 봐야함.  
같은 글자 "은행"인데, 주변 단어에 따라 뜻이 바뀜


### LLM

Embedding만으로는 "고양이"라는 좌표는 알지만, 이 문장 안에서 고양이가 누구를 가리키는지, 어떤 맥락인지 모름

<br>

✅ 정리 

Attention이 ***지금 이 단어, 문장의 어디를 참고할까?***를 정해줌.

<br>

## 비유: 회의실에서 고개 돌리기

5명이 회의실에 앉아 있다고 가정
```text
[나]   [철수]  [영희]  [민수]  [희준]
```

***나***가 말할 차례
누구 말을 가장 귀 기울여 들을까?
* 희준이가 방금 "예산" 이야기를 했다 -> 희준이에게 **많이** 귀 기울임
* 민수는 아무 말도 안 했다 -> **거의** 안 봄
* 영희가 "일정을 말했다 -> **조금** 봄

<br>

Attention도 비슷함
* 각 단어가 "지금 나한테 중요한 단어는 누구지?"를 계산
* 중요한 단어일수록 **더 많이 참고**
* 덜 중요한 단어는 **덜 참고**

이 비율을 ***가중치(weight)*** 라고 부름.  
"희준 70%, 영희 20%, 나머지 10%"처럼

<br>

## 쉬운 예시

문장: "고양이가 쥐를 잡았다"

"잡았다"를 이해할 때 Attention은 대략 이렇게 동작

```text
잡았다 → 고양이(많이) + 쥐(많이) + 가(조금) + 를(조금)
```

* 누가 잡았는지 -> 고양이
* 무엇을 잡았는지 -> 쥐

✅ 정리  
"잡았다"는 혼자서는 불완전하고, 고양이와 쥐를 같이 봐야 문장이 완성

<br>

## Query, Key, Value

| 이름 | 쉬운 뜻 | 비유 |
|------|---------|------|
| Query (Q) | "나는 지금 뭘 찾고 있지?" | 도서관에서 던지는 질문 |
| Key (K) | "나는 이런 정보를 가지고 있어" | 책 표지에 붙은 꼬리표 |
| Value (V) | "실제 내용은 이거야" | 책 안의 내용 |
  
흐름:
1. 내 단어가 질문(Query)을 던짐 -> "나랑 관련된 단어 누구?"
2. 다른 단어들의 꼬리표(Key)와 비교
3. 잘 맞는 단어일수록 내용(Value)을 더 많이 가져옴
  
도서관 비유:
* "고양이에 관한 책 있어요?" (Query)
* 사서가 책 꼬리표(Key)를 훑어봄
* 맞는 책의 내용(Value)을 가져다 줌

<br>

## Self-Attention?

Attention 중에서도, ***같은 문장 안의 단어들끼리*** 서로를 참고하는 것.

일반 Attention: A가 B를 참고 (다른 문장, 다른 모달리티 등)  
Self-Attention: A가 ***자기 문장 안의*** 다른 단어들만 참고

```text
문장: [고양이] [가] [쥐를] [잡았다]

"잡았다"의 Query → 같은 문장의 [고양이, 가, 쥐를, 잡았다] Key들과 비교
```

회의실 비유로 말하면, ***같은 회의실에 앉은 5명끼리만*** 고개를 돌리는 것.  
밖에서 온 사람(다른 문장)은 안 봄.

<br>

### 예시 1: "고양이가 쥐를 잡았다"

위 쉬운 예시가 바로 Self-Attention.

"잡았다"가 Self-Attention 할 때:

| 단어 | 가중치 (대략) | 이유 |
|------|--------------|------|
| 고양이 | 높음 | 누가 잡았는지 |
| 쥐를 | 높음 | 무엇을 잡았는지 |
| 가, 를 | 낮음 | 문법 조사, 의미 기여 적음 |

각 단어가 ***자기 문장 안***의 다른 단어를 보고 가중치를 정함.

<br>

### 예시 2: 같은 글자, 다른 뜻 (은행)

**문장 A:** "은행에서 돈을 찾았다"

```text
"돈"이 Self-Attention 할 때:
  → "은행" (많이) + "찾았다" (많이) + "에서" (조금)
  → "돈 + 은행" → 금융기관 맥락
```

**문장 B:** "강가에서 은행을 걸었다"

```text
"은행"이 Self-Attention 할 때:
  → "강가" (많이) + "걸었다" (많이) + "에서" (조금)
  → "은행 + 강가" → 나무 맥락
```

같은 "은행"이라도 ***주변 단어(같은 문장 안)*** 에 따라 참고하는 대상이 달라짐.  
Embedding만으로는 안 되고, Self-Attention이 맥락을 붙여 줌.

<br>

### 예시 3: 각 단어가 "업데이트"되는 과정

Self-Attention의 결과 = ***각 단어의 새 표현***

```text
입력:  고양이  가  쥐를  잡았다
         ↓    ↓    ↓     ↓
출력:  고양이' 가'  쥐를'  잡았다'
```

* `고양이'`: "나는 이 문장에서 주어, 쥐를 잡는 존재"라는 정보가 섞임
* `쥐를'`: "나는 잡히는 대상"이라는 정보가 섞임
* `잡았다'`: "고양이가 쥐를 잡았다"가 한꺼번에 반영됨

같은 문장의 Value들을 가중합해서 각 단어를 새로 만드는 것이 Self-Attention의 출력.

<br>

### 예시 4: Q, K, V가 모두 같은 문장에서 나옴

도서관 비유를 Self-Attention에 맞게 바꾸면:

```text
문장 = 도서관 한 구역의 책들

"잡았다" (Query):  "나랑 관련된 단어 누구지?"
"고양이" (Key):    "나는 주어/행위자 정보를 가짐"
"쥐를" (Key):      "나는 목적어/피동작 대상 정보를 가짐"
"가" (Key):        "나는 조사야, 별로 안 중요해"

→ Key가 Query와 잘 맞으면 → 그 단어의 Value(실제 의미 벡터)를 많이 가져옴
```

Self-Attention의 핵심: ***Q, K, V가 전부 같은 문장의 단어들에서 나온다***

<br>

### Self-Attention vs Cross-Attention

| | Self-Attention | Cross-Attention |
|--|----------------|-----------------|
| Query 출처 | 이 문장 | 이 문장 (또는 디코더) |
| Key/Value 출처 | ***같은 문장*** | ***다른 문장*** |
| 예시 | "잡았다"가 "고양이·쥐" 참고 | 번역 시 한국어가 영어 원문 참고 |

Transformer 인코더는 Self-Attention만 쓰고,  
번역기 디코더는 Self-Attention + Cross-Attention을 같이 씀.

<br>

✅ 정리

> Self-Attention = 문장 안의 각 단어가, 같은 문장의 다른 단어들에게 "너랑 나 얼마나 관련 있어?"를 물어보고, 관련 있는 단어의 정보를 비율대로 섞어서 자기 자신을 새로 표현하는 것

<br>

## 전체 흐름에서의 위치

```text
문장 입력
   ↓
Tokenization (글자/단어를 조각으로 나눔)
   ↓
Embedding (각 조각을 의미 좌표로 변환)
   ↓
Attention (단어들이 서로를 얼마나 볼지 결정)  ← 지금 여기
   ↓
Transformer (Attention을 여러 번 반복)
   ↓
다음 단어 예측
```

✅ 정리

> Embedding이 각 단어의 지도 좌표를 만든다면,  
> Attention은 지금 이 좌표를 해석할 때, 어떤 다른 좌표를 같이 볼지 정하는 단계

<br>

## 내가 이해한 내용

Attention은 단어가 문장 안 다른 단어를 얼마나 참고할지 정하는 단계.  
가중치에 따라서 중요한 단어인지, 덜 중요한 단어인지 알 수 있음.  
  
Self-Attention은 같은 문장 안에서 단어들끼리 서로를 보는 것.  
Query, Key, Value 모두 같은 문장에서 나옴.  
  
Cross-Attention은 Query의 출처와 Key, Value의 출처가 다름.  
  
Query, Key, Value를 사용하기 때문에 옛날방식 (RNN 등)의 단점을 보완함.  
-> **맥락, 멀리 있는 단어 연결, 같은 글자 다른 뜻** 처리 가능

<br>

## 아직 헷갈리는 부분

### Attention vs Self-Attention — 포함 관계

```text
Attention (큰 개념: Q·K·V로 "얼마나 참고할지" 계산)
├── Self-Attention   — Q, K, V 모두 같은 문장
└── Cross-Attention  — Q와 K/V 출처가 다름
```

* **Attention** = 다른 정보를 얼마나 참고할지 정하는 메커니즘 전체
* **Self-Attention** = 그중 **같은 문장 안**에서만 서로 참고하는 경우
* 노트 앞부분 "고양이가 쥐를 잡았다" 예시 → Attention이라고 부르지만, 실제로는 Self-Attention
* LLM 맥락에서 "Attention"만 말할 때 → 대부분 Self-Attention을 가리킴

<br>

세 가지를 한눈에 비교

| | RNN (옛날 방식) | Self-Attention | Cross-Attention |
|--|----------------|----------------|-----------------|
| 읽는 방식 | 왼쪽→오른쪽 순서대로 | 문장 안 모든 단어를 동시에 비교 | 한쪽이 다른 쪽을 참고 |
| 기억 방식 | 고정 크기 hidden state 1개 | 필요한 단어를 직접 골라 봄 | 필요한 단어를 **다른 문장**에서 골라 봄 |
| 참고 범위 | 지금까지 읽은 것 (점점 희미해짐) | **같은 문장** 안 | **다른 문장** (원문, 이미지 등) |
| Q, K, V | 없음 | Q, K, V 모두 같은 문장 | Q는 한쪽, K/V는 다른 쪽 |

<br>

### RNN — 왜 "앞부분을 잊기 쉬웠나?"

```text
고양이 → 가 → 쥐를 → 잡았다
  h₁  →  h₂  →  h₃  →  h₄   (기억상자 하나를 계속 업데이트)
```

* 문장을 **한 단어씩** 읽으며 작은 기억상자(hidden state)에 압축
* 길어질수록 **앞 단어 신호가 약해지고**, 최근 단어에 더 의존
* 예: "철수는 … (중간 20문장) … **그**는 잡았다" → "그"가 누구인지 헷갈리기 쉬움

<br>

### Self-Attention — 같은 문장 안에서

```text
"잡았다"(Q) → [고양이, 가, 쥐를, 잡았다](K) 비교 → 관련 단어의 Value 가져옴
```

* **거리와 상관없이** 문장 안 어디든 직접 참고 가능
* 같은 글자 다른 뜻("은행")도 **주변 단어**와 같이 보면 해결
* GPT처럼 한 문장만 다루는 모델은 이것만 사용

<br>

### Cross-Attention — 다른 문장(또는 다른 입력)에서

```text
한국어 "잡았다"(Q) → 영어 [cat, caught, mouse](K) 비교
```

* Self-Attention과 **계산은 같고**, Q와 K/V의 **출처만 다름**
* 번역·요약: 디코더(출력)가 인코더(원문)를 참고
* 이미지+텍스트: 텍스트가 이미지 패치를 참고
* 번역기 디코더는 **Self-Attention + Cross-Attention** 둘 다 사용

<br>

### 헷갈릴 때 이렇게 구분

```text
같은 문장 안만 보나?  → Self-Attention
다른 문장/입력을 보나? → Cross-Attention
순서대로만 읽나?      → RNN
```

✅ 정리

> RNN은 순서대로 읽으며 기억이 희미해지는 문제가 있었고,  
> Self-Attention은 **같은 문장** 안에서 멀리 있는 단어도 직접 연결하고,  
> Cross-Attention은 **다른 문장**을 참고할 때 쓴다.

# 06. Inference & Generation

<br>

## 한 줄 설명

Inference & Generation = Next Token Prediction으로 만든 **확률표**를 보고, **실제로 토큰을 고르고** 반복해서 **답변 전체를 완성**하는 것

<br>

## 이전 노트와의 연결

05번에서 배운 **Next Token Prediction**은 "다음에 올 말" 후보들에게 **확률(%)**을 매기는 단계.

```text
"다"  45%
"."   20%
"고"   5%
...
```

**Inference & Generation**은 그다음 단계:

```text
Next Token Prediction  →  확률표 만들기
Inference              →  확률표 보고 실제로 1개 고르기
Generation             →  고른 걸 붙이고 반복해서 답 완성
```

비유:

* **Next Token Prediction** = 시험지에 "다음 빈칸 후보와 확률"을 적는 것
* **Inference & Generation** = 그중 하나를 골라 쓰고, 빈칸을 계속 채워 **글 전체를 완성**하는 것

<br>

✅ 정리

> Next Token Prediction이 **"확률을 만드는 엔진"**이라면,  
> Inference & Generation은 그 엔진을 **실제로 돌려서 답변을 완성하는 방법**

<br>

## Inference vs Generation

| 용어 | 쉬운 뜻 | 비유 |
|------|---------|------|
| **Inference (추론)** | 이미 배운 모델을 **실제로 사용**하는 것 | 연습 끝난 학생이 **시험장에 들어가 문제 푸는 것** |
| **Generation (생성)** | 추론 결과로 **새 글을 만들어 내는 것** | 시험에서 **직접 답안지에 글을 쓰는 것** |

학습(Training) 때는 정답이 있었지만,  
Inference & Generation 때는 **정답 없이** 모델이 스스로 글을 이어 씀.

<br>

## 한 장면으로 보기

사용자가 ChatGPT에 이렇게 물었다고 가정:

```text
"고양이에 대해 짧게 설명해줘"
```

모델 안에서는 이런 일이 반복됨:

```text
  ┌─ 프롬프트 ─────────────────────────────────────────────┐
  │  "고양이에 대해 짧게 설명해줘"                            │
  └────────────────────────────────────────────────────────┘
         │
         ▼  ┌──────────┐     ┌──────────┐     ┌──────────┐
  1회차 ────│Transformer│────▶│ 확률표   │────▶│ "고" 선택 │
            └──────────┘     └──────────┘     └────┬─────┘
                                                   │ 붙이기
         ┌─────────────────────────────────────────┘
         ▼
  2회차   "...설명해줘 고"  ──▶ Transformer ──▶ "양" 선택 ──▶ 붙이기
         ▼
  3회차   "...설명해줘 고양" ──▶ Transformer ──▶ "이" 선택 ──▶ 붙이기
         ▼
        ...  (수십~수백 번 반복) ...
         ▼
  ┌─ 완성 ─────────────────────────────────────────────────┐
  │  "고양이는 작고 귀여운 포유류입니다."                      │
  └────────────────────────────────────────────────────────┘
```

**한 토큰 고르기 → 붙이기 → 다시 예측 → 반복**

* 매번 모델을 돌려 확률을 구하는 과정 = **Inference**
* 고른 토큰을 붙여 답을 완성하는 반복 = **Generation**

이 반복 방식을 **Autoregressive(자기회귀) 생성**이라고 함.

* **Auto** = 스스로
* **Regressive** = 방금 만든 걸 다시 입력으로 씀

**비유:** 타자 치기. 한 글자 치고 → 그걸 보고 → 다음 글자 고르고 → 반복

<br>

## 핵심: "확률 1등만 고를까?"

모델이 만든 확률표:

```text
"다"  45%
"."   20%
"고"   5%
...
```

**어떤 규칙으로 고를지**가 Inference & Generation의 핵심.

<br>

### Greedy (탐욕적 선택)

**항상 확률 1등만 고름**

```text
  확률표                    Greedy 선택
  ─────────────────────────────────────
  "다"  ████████████████████  45%  ◀── 항상 이것만!
  "."   ████████              20%
  "고"  ██                     5%
```

| | |
|--|--|
| 장점 | 빠르고, 비슷한 질문에 비슷한 답 |
| 단점 | 답이 **뻔하고 반복적**일 수 있음 |

**비유:** 메뉴판에서 **항상 1등 인기 메뉴만** 주문

<br>

### Sampling (확률에 따라 랜덤 선택)

**확률이 높을수록 자주 뽑히지만, 가끔 2등·3등도 선택**

```text
  확률표                    Sampling (룰렛)
  ─────────────────────────────────────
  "다"  ████████████████████  45%  ← 자주 걸림
  "."   ████████              20%  ← 가끔 걸림
  "고"  ██                     5%  ← 드물게 걸림

  🎲 매번 룰렛을 돌려서 확률에 따라 랜덤 선택
```

| | |
|--|--|
| 장점 | 답이 **다양**해짐 |
| 단점 | 가끔 **엉뚱한 말**이 나올 수 있음 |

**비유:** 주사위가 있는 룰렛 — 넓은 칸일수록 잘 걸림

<br>

✅ 정리

> **Greedy** = 항상 1등만 / **Sampling** = 확률에 따라 랜덤  
> Next Token Prediction = 확률표를 만드는 것까지  
> 실제로 뭘 고를지 = Inference & Generation

<br>

## 답의 "성격"을 조절하는 다이얼

ChatGPT 설정이나 API에서 보이는 이름들. **조절 버튼**으로 이해하면 됨.

<br>

### Temperature (온도)

확률 차이를 **얼마나 극단적으로 만들지** 조절.

```text
  온도 낮음 (0.3) — 차분, 안정          온도 높음 (1.5) — 자유, 다양
  ──────────────────────────          ──────────────────────────
  "다"  ██████████████████████ 70%    "다"  ████████████      30%
  "."   ████                  15%    "."   ██████████        25%
  "고"  █                      2%    "고"  ██████            15%
        ▲ 1등이 압도적                    ▲ 확률이 평평해짐
```

| 온도 | 느낌 | 쓰는 경우 |
|------|------|-----------|
| **낮음** | 신중, 일관적 | 사실 설명, 번역, 코드 |
| **높음** | 창의적, 변화 많음 | 글쓰기, 아이디어 브레인스토밍 |

**비유:** 물 온도 — 차가우면 움직임 적고, 뜨거우면 활발함

<br>

### Top-k

**상위 k개 후보만 남기고 나머지는 버림**

```text
Top-k = 3 이면
  "다" 45%, "." 20%, "고" 5% 만 남기고
  나머지 수만 개 후보는 제외
```

**비유:** 오디션에서 **상위 3명만** 다음 라운드로

<br>

### Top-p (Nucleus)

**확률 합이 p%가 될 때까지 상위 후보만 남김**

```text
Top-p = 0.9 (90%) 이면
  "다" 45% + "." 20% + "고" 5% + ... = 90% 될 때까지 포함
```

* Top-k = "몇 명"으로 자름
* Top-p = "확률 몇 %까지"로 자름

**비유:** 인기 투표에서 **득표율 90%가 모일 때까지** 후보를 받아들임

<br>

## 언제 멈출까?

무한히 글을 쓸 수는 없으니 **멈추는 조건**이 있음.

| 조건 | 뜻 | 비유 |
|------|-----|------|
| **Stop token** | "여기서 끝" 신호 토큰 | 마침표 `.` 또는 `</s>` 같은 종료 표시 |
| **Max length** | 최대 길이 제한 | "500자 넘으면 그만" |
| **EOS (End of Sequence)** | 문장 끝 표시 | 책의 "끝" 표시 |

```text
"고양이는 귀엽습니다." + [끝 토큰] → 생성 종료
```

<br>

## 스트리밍 — 왜 한 글자씩 보이나?

ChatGPT가 답을 **타이핑하듯** 보여주는 이유:

```text
  모델 (생성)                         화면 (사용자가 보는 것)
  ─────────────────────────────────────────────────────────

  "고" 생성 ──────────────────────▶  고
  "양" 생성 ──────────────────────▶  고양
  "이" 생성 ──────────────────────▶  고양이
  "는" 생성 ──────────────────────▶  고양이는
   ...                                ...
  "." 생성  ──────────────────────▶  고양이는 귀엽습니다.

  ❌ 전체 완성 후 한꺼번에 표시 (X)
  ✅ 토큰 1개 만들 때마다 바로 표시 (O)  ← 스트리밍
```

**비유:** 요리사가 요리를 **한 접시씩** 내놓는 것 (전부 다 만들고 한 번에 내는 게 아님)

<br>

## 학습 vs Inference

```text
  🎓 학습                              💬 Inference
  ┌────────────────────┐              ┌────────────────────┐
  │ 정답 ✅ 있음        │              │ 정답 ❌ 없음         │
  │ 모델 키우기          │              │ 답 만들기            │
  │ 입력: 정답 전체      │              │ 입력: 지금까지 쓴 것  │
  │ 계산: 문장당 1번     │              │ 계산: 토큰당 1번     │
  │ 비유: 연습문제       │              │ 비유: 시험장         │
  └────────────────────┘              └────────────────────┘
```

| | 학습 (Training) | Inference & Generation |
|--|-----------------|------------------------|
| 정답 | 있음 | 없음 |
| 하는 일 | 모델을 똑똑하게 키움 | 똑똑해진 모델로 답 만들기 |
| 문장 입력 | 정답 전체를 한꺼번에 | 지금까지 쓴 것만 |
| 반복 | 문장 1개 = 계산 1번 | 토큰 1개 = 계산 1번 |
| 비유 | 정답지 있는 연습 | 시험장에서 직접 쓰기 |

<br>

## 비유: 스마트폰 자동완성 (강화 버전)

1. 내가 "내일 날씨가"라고 입력 (프롬프트)
2. 키보드가 후보와 확률 계산 (Inference)
3. "좋"을 고름 (Generation 1회)
4. "내일 날씨가 좋"을 보고 또 계산
5. "을", "것", " 같", "습니다" … 반복
6. 문장이 완성되거나 길이 제한에 도달하면 멈춤

ChatGPT는 이 과정을 **초당 수십~수백 토큰**으로 하고,  
Temperature·Top-p 같은 설정으로 **답의 성격**을 조절함.

<br>

## 전체 흐름에서의 위치

```text
  ┌─────────────┐   ┌───────────┐   ┌──────────┐   ┌─────────────┐
  │Tokenization │ → │ Embedding │ → │ Attention│ → │ Transformer │
  │ 글 → 토큰   │   │ 의미 부여  │   │ 서로 참고 │   │ 깊이 이해    │
  └─────────────┘   └───────────┘   └──────────┘   └──────┬──────┘
                                                            ▼
                                                   ┌─────────────────┐
                                                   │  Next Token     │
                                                   │  확률표 만들기   │
                                                   └────────┬────────┘
                                                            ▼
                          ╔═══════════════════════════════════════════╗
                          ║  Inference  →  확률에서 1개 고르기  ← 여기! ║
                          ║  Generation →  붙이고 반복         ← 여기! ║
                          ╚═══════════════════════════════════════════╝
                                                            ▼
                                                   ┌─────────────────┐
                                                   │  완성된 답변     │
                                                   └─────────────────┘
```

<br>

## 핵심 정리

1. **Inference** = 배운 모델을 실제로 돌려 "다음 말" 확률을 구하는 것
2. **Generation** = 확률에서 토큰 1개를 골라 붙이는 것을 반복
3. **Greedy** = 항상 확률 1등 / **Sampling** = 확률에 따라 랜덤 선택
4. **Temperature** = 답의 다양성 조절 (낮음 = 안정, 높음 = 창의)
5. **Top-k / Top-p** = 후보를 줄여서 엉뚱한 선택 방지
6. **Stop token / Max length** = 생성 종료 조건
7. **스트리밍** = 토큰을 만들 때마다 바로 화면에 표시

<br>

✅ 한 문장으로

> **Inference는 배운 모델을 실제로 돌려 "다음 말" 확률을 구하고, Generation은 그중 하나를 골라 붙이는 일을 반복해서 ChatGPT 답변 전체를 만든다.**

<br>

## 다음 단계

다음 학습 주제는 **RAG (Retrieval-Augmented Generation)**.

* 모델이 글을 쓰기 **전에** 참고 자료를 찾아오는 방법
* 외부 문서·DB를 검색해서 답에 반영하는 구조
* Inference & Generation 위에 "검색" 단계가 추가되는 것

Inference & Generation이 **"어떻게 쓸지"**라면,  
RAG는 **"무엇을 참고해서 쓸지"**에 가까움.

<br>

## 내가 이해한 내용

Next Token Prediction으로 확률표를 만든 뒤, Greedy나 Sampling으로 토큰 하나를 고르고 붙인다.  
이걸 Stop token이나 max length에 도달할 때까지 반복하면 ChatGPT 답변이 완성된다.  
Temperature와 Top-p는 "얼마나 안전하게/창의적으로 고를지"를 조절하는 다이얼이다.

<br>

### 학습 vs Inference — 뭐가 다른 거야?

둘 다 Transformer를 돌리고 "다음 토큰"을 예측한다는 점은 같음.  
**목적·입력 방식·반복 횟수**가 완전히 다름.

```text
┌──────────────────────────────┐       ┌──────────────────────────────┐
│     🎓 학습 (Training)      │       │     💬 Inference (ChatGPT)   │
│     "모델 키우기"             │       │     "답 만들기"               │
├──────────────────────────────┤       ├──────────────────────────────┤
│                              │       │                              │
│  ┌────────────────────────┐  │       │  ┌────────────────────────┐  │
│  │  정답 문장 전체 넣기     │  │       │  │  지금까지 쓴 글만 넣기   │  │
│  │  [고양이][가][귀][엽][다]│  │       │  │  [고양이] → 점점 늘어남  │  │
│  └───────────┬────────────┘  │       │  └───────────┬────────────┘  │
│              ▼               │       │              ▼               │
│  ┌────────────────────────┐  │       │  ┌────────────────────────┐  │
│  │  Transformer × 1번     │  │       │  │  Transformer × 1번     │  │
│  └───────────┬────────────┘  │       │  └───────────┬────────────┘  │
│              ▼               │       │              ▼               │
│  ┌────────────────────────┐  │       │  ┌────────────────────────┐  │
│  │  모든 위치 동시 예측    │  │       │  │  마지막 위치만 예측     │  │
│  │  위치0 위치1 위치2 ...  │  │       │  │  ────────────▶ ?       │  │
│  └───────────┬────────────┘  │       │  └───────────┬────────────┘  │
│              ▼               │       │              ▼               │
│  ┌────────────────────────┐  │       │  ┌────────────────────────┐  │
│  │  정답과 비교 → 수정     │  │       │  │  토큰 1개 고르기 → 붙임 │  │
│  │  (가중치 업데이트)      │  │       │  └───────────┬────────────┘  │
│  └────────────────────────┘  │       │              │               │
│                              │       │         끝?──┴──┐            │
│         ✅ 1번에 끝           │       │          │  아니오          │
│                              │       │          ▼      │            │
│                              │       │       답변완성   └─ 반복 ─┘   │
└──────────────────────────────┘       └──────────────────────────────┘
```

| 비교 항목 | 학습 (Training) | Inference (추론) |
|-----------|-----------------|------------------|
| **목적** | 모델을 똑똑하게 **키우기** | 똑똑해진 모델로 **답 만들기** |
| **정답** | 있음 (정답 문장 제공) | 없음 (스스로 선택) |
| **입력** | 정답 문장 **전체**를 한꺼번에 | **지금까지 쓴 것**만 (프롬프트 + 생성된 글) |
| **예측 위치** | 문장 안 **모든 위치** 동시에 | **마지막 위치**만 |
| **Transformer 실행** | 문장 1개 = **1번** | 토큰 1개 = **1번** (수십~수백 번 반복) |
| **결과 활용** | 틀린 부분 고쳐서 **가중치 수정** | 고른 토큰을 **답변에 붙임** |
| **비유** | 정답지 있는 연습문제 | 시험장에서 한 글자씩 직접 쓰기 |

<br>

### 예시 1 — 같은 문장, 완전 다른 방식

문장: **"고양이가 귀엽다"**  
토큰: `[고양이] [가] [귀] [엽] [다]`

<br>

#### 🎓 학습할 때 — Teacher Forcing

정답 문장을 **처음부터 끝까지 한꺼번에** 넣음.  
Transformer를 **딱 1번** 돌리면, 모든 위치의 예측이 **동시에** 나옴.

```text
  ══════════ STEP 1: 정답 문장 전체를 한꺼번에 입력 ══════════

       ┌────────┬────┬────┬────┬────┐
       │ 고양이 │ 가 │ 귀 │ 엽 │ 다 │  ← 정답 전체 (Teacher Forcing)
       └────────┴────┴────┴────┴────┘
                         │
                         ▼
              ┌─────────────────────┐
              │   Transformer ×1    │  ← 딱 1번만 실행!
              └──────────┬──────────┘
                         ▼
  ══════════ STEP 2: 모든 위치에서 동시에 예측 ══════════

       ┌────────┐   ┌────┐   ┌────┐   ┌────┐   ┌────┐
       │ 고양이 │   │ 가 │   │ 귀 │   │ 엽 │   │ 다 │
       └───┬────┘   └─┬──┘   └─┬──┘   └─┬──┘   └─┬──┘
           │          │        │        │        │
           ▼          ▼        ▼        ▼        ▼
         "가"?      "귀"?    "엽"?    "다"?      ?
           ✅         ✅       ✅       ✅
           │          │        │        │
           └──────────┴────────┴────────┘
                         │
                         ▼
  ══════════ STEP 3: 정답과 비교 후 모델 수정 ══════════

              ┌─────────────────────┐
              │  Cross-Entropy Loss │
              │  틀린 확률 → 수정    │
              └─────────────────────┘

  💡 빈칸 채우기 비유:
     "고양이 _ 가 _ 귀 _ 엽 _ 다"  →  4개 빈칸을 동시 채점
```

<br>

#### 💬 Inference할 때 — Autoregressive 생성

정답 없음. **한 토큰씩** 고르고 붙이며 **Transformer를 반복** 실행.

```text
  글이 한 토큰씩 "자라나는" 과정 (정답 없음, 순서대로만 진행)

  1회차   ┌────────┐                    ┌────┐
          │ 고양이 │  ── Transformer ──▶│ 가 │ ✅ 선택
          └────────┘                    └──┬─┘
                                             │ 붙이기
  2회차   ┌────────┬────┐                 ┌────┐
          │ 고양이 │ 가 │ ── Transformer ─▶│ 귀 │ ✅
          └────────┴────┘                 └──┬─┘
                                             │ 붙이기
  3회차   ┌────────┬────┬────┐            ┌────┐
          │ 고양이 │ 가 │ 귀 │ ── T ──────▶│ 엽 │ ✅
          └────────┴────┴────┘            └──┬─┘
                                             │ 붙이기
  4회차   ┌────────┬────┬────┬────┐       ┌────┐
          │ 고양이 │ 가 │ 귀 │ 엽 │ ── T ─▶│ 다 │ ✅
          └────────┴────┴────┴────┘       └──┬─┘
                                             │ 붙이기
  5회차   ┌────────┬────┬────┬────┬────┐
          │ 고양이 │ 가 │ 귀 │ 엽 │ 다 │ ── T ──▶ [END] 🏁
          └────────┴────┴────┴────┴────┘

  ════════════════════════════════════════════════════════
  핵심: 입력 박스가 매 회차 1칸씩 늘어남  +  Transformer도 매번 1번씩
  ════════════════════════════════════════════════════════
```

**핵심 차이 — 계산 횟수:**

```text
  같은 5토큰 문장 "고양이가 귀엽다"

  🎓 학습                          💬 Inference
  ┌────────────────────┐          ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐
  │  Transformer ×1    │          │T │→│T │→│T │→│T │→│T │
  │  5위치 동시 예측    │          └──┘ └──┘ └──┘ └──┘ └──┘
  └────────────────────┘           1회  2회  3회  4회  5회

       1번                            4~5번
```

### 예시 2 — ChatGPT 대화로 보기

사용자: **"고양이에 대해 한 문장으로 설명해줘"**

<br>

#### 🎓 이 문장으로 학습했다면 (Training)

수억 개 문장 중 하나가 이런 정답이었을 수 있음:

```text
정답: "고양이는 작고 귀여운 포유류입니다."
```

학습 시:

```text
  ⚠️ 사용자 질문은 이 과정에 없음 (정답 문장만 학습)

  ┌─ 정답 문장 전체 입력 ──────────────────────────────────────┐
  │ [고양이는][작고][귀여운][포유류입니다][.]                    │
  └────────────────────────────┬───────────────────────────────┘
                               ▼
                    ┌─────────────────────┐
                    │  Transformer × 1    │
                    └──────────┬──────────┘
                               ▼
              모든 위치 동시 예측 → 정답 비교 → 모델 수정
```

<br>

#### 💬 ChatGPT가 실제로 답할 때 (Inference)

```text
  ╔══════════════════════════════════════════════════════════╗
  ║              💬 Inference 생성 루프 (반복)                ║
  ╚══════════════════════════════════════════════════════════╝

       ┌─ 프롬프트 ──────────────────────────────────────┐
       │ "고양이에 대해 한 문장으로 설명해줘"               │
       └──────────────────────┬──────────────────────────┘
                              ▼
       ╭──────────────────────────────────────────────────╮
       │  ① 현재까지의 글 (프롬프트 + 생성된 토큰)          │
       │                        ▼                         │
       │  ② Transformer × 1  →  ③ LM Head (확률표)       │
       │                        ▼                         │
       │  ④ Greedy / Sampling → 토큰 1개 선택              │
       │                        ▼                         │
       │  ⑤ 선택한 토큰을 글에 붙이기                       │
       │                        ▼                         │
       │         끝? ──── 아니오 ──────────┐              │
       │          │                        │              │
       │         예 (Stop/max)              │              │
       ╰──────────┼────────────────────────┼──────────────╯
                  ▼                        │
           ┌────────────┐                  │
           │  답변 완성  │                  │
           └────────────┘                  │
                                           └──▶ ① 로 돌아감

  글이 자라나는 모습:
  ┌──────────┐   ┌──────────┐   ┌────────────┐   ┌──────────────┐
  │ 프롬프트  │ → │ ...고    │ → │ ...고양이는 │ → │ ...입니다.  │
  └──────────┘   └──────────┘   └────────────┘   └──────────────┘
     시작           1토큰          2토큰              완성
```

### 왜 학습은 1번, Inference는 여러 번?

```text
  🎓 학습 — 왜 1번이면 되나?              💬 Inference — 왜 여러 번?

  ┌──────────────────────┐              ┌──────────────────────┐
  │ 📋 정답 문장 전체 있음 │              │ ❓ 정답 없음            │
  └──────────┬───────────┘              └──────────┬───────────┘
             ▼                                     ▼
  ┌──────────────────────┐              ┌──────────────────────┐
  │ 미래 토큰도 입력에 포함 │              │ 다음 토큰을 먼저 골라야 │
  │ (Teacher Forcing)     │              │ 고른 뒤에야 다음 예측   │
  └──────────┬───────────┘              └──────────┬───────────┘
             ▼                                     ▼
  ┌──────────────────────┐              ┌──────────────────────┐
  │ 모든 위치 병렬 계산    │              │ 순서대로 1개씩만 가능   │
  │ GPU 한 번에 처리      │              │ "가" 전엔 "귀" 불가    │
  └──────────┬───────────┘              └──────────┬───────────┘
             ▼                                     ▼
       Transformer ×1                        Transformer ×N
```

**비유 — 레고 조립:**

```text
  🎓 학습 (연습)                         💬 Inference (실전)

  ┌─────────┐                           ┌───┐
  │완성품   │ 사진 보며                  │ 1 │ 조각부터
  │  사진   │ 4개 빈칸 동시 채점          └─┬─┘
  └─────────┘                              ▼
       1번                                  ┌───┐
                                           │ 2 │ 붙이고
                                           └─┬─┘
                                             ▼
                                            ...반복...
                                             ▼
                                           ┌─────────┐
                                           │ 완성품  │
                                           └─────────┘
```

### Causal Mask — 학습·Inference 둘 다 적용

둘 다 **미래 토큰을 못 보게** 막음. 이건 학습과 Inference가 **같은 규칙**을 쓴다는 뜻.

```text
  Causal Mask — "미래는 못 본다" (학습·Inference 공통)

              볼 수 있는 범위 (삼각형으로 늘어남)
              ─────────────────────────────────
                    고양이   가   귀   엽   다
              고양이   ✅
              가       ✅   ✅
              귀       ✅   ✅   ✅    ← "귀" 위치: 엽, 다는 ❌
              엽       ✅   ✅   ✅   ✅
              다       ✅   ✅   ✅   ✅   ✅

              ▲ 왼쪽(과거)만 보임   ▼ 오른쪽(미래)는 가림
```

### 한눈에 비교 — 타임라인

같은 문장 `"고양이가 귀엽다"` 기준:

```text
  같은 문장 "고양이가 귀엽다" — 시간 흐름 비교

  🎓 학습: 시간 ──────────────────────────────────────────▶

     ┌─────────────────────────────────────────────────────────┐
     │ [고양이][가][귀][엽][다]  →  T×1  →  5위치예측  →  수정  │
     └─────────────────────────────────────────────────────────┘
                                    🏁 끝 (1스텝)


  💬 Inference: 시간 ────────────────────────────────────────▶

     ┌────────┐      ┌──────────┐      ┌────────────┐
     │[고양이]│─T×1─▶│[고양이][가]│─T×1─▶│[고양이][가][귀]│─T×1─▶ ...
     └───┬────┘      └────┬─────┘      └──────┬─────┘
         "가"              "귀"                  "엽"
                                                        │
     ... ─T×1─▶ ┌──────────────────────┐ ─T×1─▶ 🏁 END   │
                │[고양이][가][귀][엽][다]│               │
                └──────────────────────┘                 │
                     "다" ◀──────────────────────────────┘
```

| | 학습 | Inference |
|--|------|-----------|
| Transformer 호출 | **1번** | **토큰 수만큼** (보통 수십~수백 번) |
| 입력 길이 | 정답 전체 (고정) | 매 스텝 1토큰씩 증가 |
| 출력 사용 | loss 계산용 | 다음 입력에 붙임 |
| 모델 변경 | ✅ 가중치 수정 | ❌ 가중치 그대로 (읽기만) |

<br>

### 자주 헷갈리는 질문 3가지

**Q1. 학습할 때도 "다음 토큰 예측" 아닌가?**  
→ 맞음. 다만 **정답 문장 전체를 넣고 모든 위치를 동시에** 예측함.  
Inference는 **마지막 위치 1곳만** 예측하고, 고른 걸 붙여 **다시** 돌림.

**Q2. Inference 때 모델이 바뀌나?**  
→ **안 바뀜.** 학습 때 키운 가중치를 **그대로 읽기만** 함.  
학습 = 모델 수정 / Inference = 모델 사용.

**Q3. 왜 ChatGPT는 느릴 때가 있나?**  
→ 답이 길수록 **Transformer를 더 많이** 돌려야 하기 때문.  
100토큰 답변 ≈ Transformer 100번 실행 (단순화한 설명; 실제로는 KV Cache 등으로 최적화).

✅ 한 문장으로 (Q&A 정리)

> **학습은 정답 문장 전체를 한 번에 넣어 모든 위치에서 동시에 맞히고 모델을 고치는 연습이고, Inference는 정답 없이 토큰을 하나씩 고르고 붙이며 Transformer를 반복 실행해서 답변을 완성하는 실전이다.**

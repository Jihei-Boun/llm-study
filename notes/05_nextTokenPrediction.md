# 05. Next Token Prediction

<br>

## 한 줄 설명

Next Token Prediction = 지금까지의 글을 보고, **다음 토큰 1개**가 무엇일지 확률로 맞히는 것

<br>

## 왜 필요한가?

Transformer 노트에서 정리한 것처럼, GPT류 LLM은 **Decoder만** 씀:

```text
[Decoder만] → 이전 글자들을 보고 다음 글자 예측
```

번역처럼 "입력 문장 → 출력 문장"을 한꺼번에 만드는 게 아니라, **항상 한 토큰씩** 이어 붙임.

```text
"고양이가"        → 다음은?  "귀" (높은 확률)
"고양이가 귀"     → 다음은?  "엽" (높은 확률)
"고양이가 귀엽"   → 다음은?  "다" (높은 확률)
```

**한 토큰 예측을 반복**하면 긴 문장·답변·코드까지 만들 수 있음.  
ChatGPT가 타이핑하듯 답하는 원리가 바로 이것.

<br>

✅ 정리

> LLM의 핵심 학습 목표 = **다음 토큰 맞히기**  
> LLM의 핵심 생성 방식 = **다음 토큰 고르기를 반복**

<br>

## 비유: 스마트폰 자동완성

키보드 자동완성과 비슷함:

```text
당신이 입력: "내일 날씨가"

키보드(모델):
  "좋" 40%
  "나" 25%
  "흐" 20%
  ...
```

차이점:
* LLM은 **훨씬 긴 문맥**을 봄
* **훨씬 큰 어휘집**(수만 개 토큰)에서 고름
* Transformer로 **문맥을 깊게 이해한 뒤**, 그 결과로 다음 토큰 확률을 계산

한 번에 한 글자만 고르지만, 그걸 수천~수만 번 반복하면 에세이·코드·대화가 됨.

<br>

## 전체 흐름에서의 위치

앞에서 배운 것들은 **"문장을 이해하는 준비"**였음.

```text
"고양이가 쥐를 잡았다"
        ↓
① Tokenization    → [고양이, 가, 쥐를, 잡았다]
        ↓
② Embedding       → 각 토큰을 벡터로
        ↓
③ Positional Enc. → 순서 정보
        ↓
④ Transformer     → 문맥 깊게 이해 (수십 층 반복)
        ↓
⑤ Next Token      → "다음 조각 맞히기"  ← 여기!
```

**비유: 시험 문제 풀기**

| 단계 | 비유 |
|------|------|
| Tokenization | 문제를 조각으로 나눔 |
| Embedding | 각 조각을 메모 카드로 바꿈 |
| Transformer | 메모 카드를 여러 번 읽으며 문장 전체 이해 |
| **Next Token** | **"다음 빈칸에 뭐가 들어가?"** 맞히기 |

✅ 정리

> Next Token Prediction = Transformer가 이해한 결과로 **"다음 단어 맞히기 게임"**을 하는 단계

<br>

## 한 장면으로 보기

**"고양이가 귀엽"** 이라고 쳤다고 가정:

```text
[1] Tokenization
    → [고양이] [가] [귀] [엽]

[2] Embedding + Positional Encoding
    → 각 토큰에 의미 + 순서 붙이기

[3] Transformer (수십 층)
    → 문장 전체 맥락 이해
    → "엽" 위치에 똑똑한 메모 생성

[4] LM Head
    → "엽" 메모를 점수표로 변환
    → "다" 85점, "." 40점, ...

[5] Softmax
    → "다" 60%, "." 25%, ...

[6] 토큰 선택 (생성 단계)
    → "다" 고름

[7] 결과
    → "고양이가 귀엽다"
```

<br>

## Transformer 출력이 뭔데?

Transformer를 다 지나면, **토큰마다 "똑똑해진 메모"**가 하나씩 생김:

```text
[고양이] → 메모 A  (문장 전체를 본 뒤의 "고양이" 이해)
[가]     → 메모 B
[쥐를]   → 메모 C
[잡았다] → 메모 D
```

이 메모는 숫자 벡터 `[0.3, -0.1, 0.5, ...]` 형태.  
**"이 토큰이 이 문장 안에서 어떤 역할인지"**를 담고 있다고 보면 됨.

**중요:** Next Token Prediction은 보통 **맨 마지막 토큰의 메모**를 씀.

```text
"고양이가 쥐를 잡았다"
                        ↑
                   이 메모 D를 보고
                   "다음에 뭐 올까?" 예측
```

왜 마지막? **"지금까지 읽은 것"의 끝**이니까, 그 다음이 뭔지 물어보는 게 자연스럽기 때문.  
(학습 때는 문장 안 **모든 위치**에서 동시에 예측)

<br>

## LM Head — 점수표 만들기

메모 D는 아직 **"고양이가 쥐를 잡았다"를 이해한 상태**일 뿐,  
**"다음 토큰 후보 점수"**는 아님.

그래서 **LM Head**라는 작은 기계에 넣음:

```text
메모 D
  ↓ LM Head
점수표: 수만 개 토큰 각각에 점수
```

**비유: 퀴즈**

```text
퀴즈: "고양이가 쥐를 잡았다" 다음에 올 말은?

후보 점수:
  "다"   → 85점
  "."    → 60점
  "고"   → 10점
  "었"   →  5점
  ... (수만 개)
```

이 점수 목록이 **logits**.  
**logits = 아직 확률이 아닌, raw 점수표**

<br>

## Softmax — 점수를 확률로 바꾸기

점수만 있으면 "85점이 높은 건 알겠는데, 정확히 몇 %?"가 애매함.

**Softmax**는 점수표를 **확률(%)**로 바꿔줌:

```text
점수:                    확률:
  "다"  85점       →      "다"  45%
  "."   60점       →      "."  20%
  "고"  10점       →      "고"   5%
  ...                    ... (합 = 100%)
```

**비유:** 시험 점수를 백분위로 환산하는 것

이제 모델은 이렇게 말하는 셈:

> "다음 토큰은 아마 '다'일 것 같아 (45%)"

**Next Token Prediction의 결과물 = 이 확률표**

<br>

✅ 정리

> Transformer가 **문맥 메모(벡터)**를 만들면,  
> LM Head가 **점수표(logits)**를 만들고,  
> Softmax가 **확률(%)**로 바꿈

<br>

## 학습할 때 — 정답이 있을 때

학습 때는 **정답 문장**이 있음:

```text
문장: "고양이가 귀엽다"

모델에게 주는 문제:
  "고양이" 뒤에 뭐?  → 정답: "가"
  "가" 뒤에 뭐?      → 정답: "귀"
  "귀" 뒤에 뭐?      → 정답: "엽"
  "엽" 뒤에 뭐?      → 정답: "다"
```

모델이 "엽" 뒤에 "다"를 30%만 줬다면?  
→ **"다" 확률을 더 높이도록** 모델을 고침.  
→ 수억 문장으로 반복하면 점점 잘 맞힘.

### Teacher Forcing

생성할 때처럼 한 글자씩 붙이며 학습하면 느림.  
그래서 학습 때는 **정답 문장 전체를 한꺼번에** 넣고,  
**모든 위치에서 동시에** "다음 토큰"을 맞히게 함.

**비유:** 빈칸 채우기 문제집에 정답이 적혀 있고, 틀린 칸만 골라 공부하는 것

### Cross-Entropy Loss

이름은 어렵지만 뜻은 단순:

> **"정답 토큰에 준 확률이 낮으면 벌점, 높으면 칭찬"**

정답이 "다"인데 모델이 "다"에 10%만 줬다 → 큰 벌점  
정답이 "다"인데 모델이 "다"에 90% 줬다 → 작은 벌점

<br>

## 생성할 때 — ChatGPT가 글 쓸 때

학습이 끝나면, **정답 없이** 글을 씀:

```text
1단계  입력: "고양이가"       → "귀" 고름
2단계  입력: "고양이가 귀"    → "엽" 고름
3단계  입력: "고양이가 귀엽"  → "다" 고름

결과: "고양이가 귀엽다"
```

**한 토큰 고르기 → 붙이기 → 다시 예측 → 반복**

이게 **Autoregressive(자기회귀) 생성**.
* **Auto** = 스스로
* **Regressive** = 방금 만든 걸 다시 입력으로 씀

**비유:** 타자 치기. 한 글자 치고 → 그걸 보고 → 다음 글자 고르고 → 반복

<br>

## Causal Masking — 미래를 못 보게 막기

```text
"고양이가 귀엽다"
```

"귀"를 예측할 때, 모델은 **"엽", "다"를 보면 안 됨**:

```text
"귀" 위치에서 볼 수 있는 것:
  ✅ 고양이, 가, 귀
  ❌ 엽, 다  ← 미래! 못 봄
```

왜? 실제로 글 쓸 때도 **아직 안 쓴 미래 글자**는 모르니까.  
미래를 미리 보면 맞히기가 아니라 **부정행위(답 안 보고 시험)**가 됨.

**Causal Mask(인과 마스크)** = 각 토큰이 **자기 앞까지만** 볼 수 있게 막는 장치

<br>

## 학습 vs 생성

| | 학습 | 생성 (ChatGPT) |
|--|------|----------------|
| 정답 | 있음 | 없음 |
| 문장 | 정답 전체를 한꺼번에 넣음 | 지금까지 쓴 것만 넣음 |
| 예측 | 모든 위치에서 동시에 | 마지막 위치에서만 |
| 목적 | 모델을 똑똑하게 키움 | 실제 답변 만들기 |
| 반복 | 한 문장 = 1번 계산 | 토큰 1개 = 1번 계산 |

**비유**
* **학습** = 정답지 있는 연습문제 여러 개 한꺼번에 풀기
* **생성** = 시험장에서 한 글자씩 직접 쓰기

<br>

## "가장 높은 확률"만 고르나?

아님. 그건 다음 주제인 **Inference & Generation**에 가까움.

```text
"다" 45%  /  "." 20%  /  "고" 5%
```

* **Greedy**: 항상 1등만 고름 → "다"
* **Sampling**: 확률에 따라 랜덤 선택 → 가끔 "."도 고를 수 있음

Next Token Prediction = **확률표를 만드는 것**까지  
**실제로 뭘 고를지** = Inference & Generation

<br>

## 이전 노트와의 연결

```text
Tokenization  → 글을 토큰으로 쪼갬
Embedding     → 토큰을 벡터로
Attention     → 문장 안에서 서로 참고
Transformer   → 위 과정을 여러 층 반복
Next Token    → 그 결과로 "다음 토큰" 확률 예측  ← 지금
Inference     → 확률에서 실제 토큰 고르고 반복   ← 다음
```

<br>

## 핵심 정리

1. **목표**: 지금까지의 토큰들을 보고 **다음 토큰 1개** 예측
2. **벡터**: Transformer가 만든 "문맥 이해 메모" (토큰마다 1개)
3. **LM Head**: 메모 → 다음 토큰 **점수표(logits)**
4. **Softmax**: 점수표 → **확률(%)**
5. **학습**: 정답과 비교해서 모델 고침 (Teacher Forcing)
6. **생성**: 확률 보고 토큰 1개 고르고 반복 (Autoregressive)
7. **제약**: Causal Mask로 **미래 토큰 참조 불가**

<br>

✅ 한 문장으로

> **Transformer가 "지금까지의 글"을 이해하고, LM Head가 "다음 후보"에 점수를 매기고, Softmax가 확률로 바꾼다. 학습은 정답과 비교하고, 생성은 하나 골라 반복한다.**

Softmax가 확률로 바꾸고, 생성은 그 중 하나를 골라 붙인다.

## 아직 헷갈리는 부분

아래는 학습 중 질문했던 내용을 다시 정리한 것. (헷갈렸던 지점 → 이해한 버전)

<br>

### LM Head가 뭐야?

Transformer 출력 = **"문장을 이해한 메모"** (벡터). 아직 "다음 토큰 점수"는 아님.

**LM Head** = 그 메모를 **어휘집 전체 토큰별 점수(logits)**로 바꾸는 마지막 출력층 (보통 `Linear` 한 층).

```text
hidden (4096)  →  LM Head  →  logits (50257)  →  Softmax  →  확률
```

* **logits** = LM Head의 출력 (아직 % 아님)
* **Softmax** = logits → 확률 (%)

<br>

### logits shape — `[batch, seq_len, vocab_size]`

3축만 기억:

| 축 | 의미 | 예 |
|----|------|-----|
| batch | 문장 몇 개 | 1 (문장 1개) |
| seq_len | 토큰 몇 개 | 5 |
| vocab_size | 후보 토큰 몇 개 | 50257 |

**한 행 = 한 위치에서 "다음 토큰" 후보별 점수표**

```text
              "가"   "귀"   "엽"   "다"   "."
위치0 (고양이뒤)  1.2   0.3   0.1   0.0   0.5   ← logits[0, 0, :]
위치1 (가 뒤)    0.2   2.1   0.4   0.1   0.0   ← logits[0, 1, :]
위치3 (엽 뒤)    0.0   0.1   0.2   8.2   3.1   ← logits[0, 3, :]  "다" 높음
```

* **행** = "여기까지 읽었을 때, **다음**에 뭐?" (퀴즈 1문제)
* **열** = 그 퀴즈의 **선택지** (후보 토큰)
* **칸** = 그 선택지에 준 점수 (logit)

⚠️ 행 이름 "엽 뒤" = `엽` **다음**을 맞히는 문제. `엽` 자체 점수가 아님.

<br>

### `logits[0, 0, :]` index 읽는 법

```python
logits[0, 0, :]
#      │  │  └─ vocab 전체 (모든 후보 점수)
#      │  └─ seq_len 위치 (0 = 첫 토큰 "고양이" 뒤)
#      └─ batch (0 = 첫 번째 문장)
```

| 코드 | 의미 |
|------|------|
| `logits[0, 0, :]` | 0번 문장, `고양이` 뒤, 후보 전체 |
| `logits[0, 3, id_다]` | 0번 문장, `엽` 뒤, `"다"` 점수 하나 |
| `logits[0, -1, :]` | 0번 문장, **마지막** 위치 (생성 시 사용) |

<br>

### 학습 — "모든 위치에서 한다"는 실제 모양

정답 문장 **전체**를 한 번에 넣음 (Teacher Forcing). Transformer **1번** 돌리면 위치마다 hidden이 **동시에** 나옴.

```text
토큰:     [고양이] [가] [귀] [엽] [다]
hidden:      h0     h1   h2   h3   h4
예측 정답:      가     귀   엽   다
```

| hidden | 본 것 | 맞혀야 할 다음 토큰 |
|--------|-------|---------------------|
| h0 | `고양이` | `가` |
| h3 | `고양이가 귀엽` | `다` |

LM Head를 **모든 hidden**에 적용 → logits shape `[1, 5, 50257]`

**학습 vs 생성**

| | 학습 | 생성 |
|--|------|------|
| 입력 | 정답 문장 전체 | 지금까지 쓴 것만 |
| forward | 문장 1개당 **1번** | 토큰 1개당 **1번** |
| LM Head | **모든** 위치 | **마지막** 위치만 |
| logits | `[1, L, V]` | `[1, 1, V]` |

<br>

### logits가 "다" 점수를 어떻게 올리나?

1. LM Head: `logits = hidden @ W^T` (W는 **학습되는** 가중치)
2. Softmax → 정답 `"다"` 확률이 낮으면 **Cross-Entropy loss** 큼
3. 역전파 → **LM Head W + Transformer** 같이 업데이트
4. `"~엽" 뒤 → "다"` 패턴을 수억 번 보면, h3과 W가 맞물려 `"다"` logit이 커짐

```text
학습 전:  "다" 15%  →  loss 큼  →  가중치 수정
학습 후:  "다" 85%  →  loss 작음
```

<br>

### shape 흐름 (코드)

```text
input_ids   [B, L]
hidden      [B, L, D]        D = d_model (4096)
logits      [B, L, V]        V = vocab_size — LM Head 출력
probs       [B, L, V]        softmax(dim=-1)

# 학습 loss: [B, L, V] → [B*L, V]로 펼쳐 labels와 cross-entropy
# 생성: hidden[:, -1:, :] → lm_head → [B, 1, V]
```

<br>

✅ 한 문장으로 (Q&A 정리)

> **logits는 LM Head 출력.** shape `[batch, 위치, 후보]` = 각 위치마다 "다음 토큰" 점수표. 학습은 정답 문장 전체를 한 번에 넣어 모든 위치에서 동시에 맞히고, loss로 정답 토큰 확률을 올린다. 생성은 마지막 위치 logits만 써서 토큰 하나 고른 뒤 반복.

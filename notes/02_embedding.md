# 02. Embedding


## 한 줄 설명

Embedding = token을 컴퓨터가 계산할 수 있는 "의미 좌표"로 바꾸는 것


## 왜 필요한가?

컴퓨터는 글자나 "고양이"라는 개념을 그대로 이해하지 못함
대신 숫자만 잘 다룸

그런데 Token ID 4521은 그냥 이름표일 뿐
-> 의미가 전혀 담겨 있지 않음(그냥 창고 선반 번호일 뿐)

| Token ID | 의미 |
|:---|:---|
| 4521 | 고양이 |
| 4522 | 강아지 |
| 9999 | 자동차 |
-> 번호만 보면 숫자 차이는 1이지만 "둘 다 동물"이라는 관계는 안 보임

Embedding
```text
4521 (고양이)  →  [0.2, -0.5, 0.8, 0.1, ...]   ← 숫자 수백~수천 개
4522 (강아지)  →  [0.3, -0.4, 0.7, 0.2, ...]   ← 비슷한 숫자 패턴
9999 (자동차)  →  [-0.9, 0.6, -0.1, 0.4, ...]  ← 꽤 다른 패턴
```
> 의미가 비슷한 단어 -> 숫자 묶음도 비슷하게 나오도록 학습됨


## 비유

### 비유 1: 지도 위의 점

각 단어를 지도 위의 한 점이라고 생각
    * 고양이와 강아지 -> 가까이 있음 (둘 다 동물)
    * 고양이와 자동차 -> 멀리 있음
    * 행복과 기쁨 -> 가까이 있음 (둘 다 긍정 감정)

(실제 embedding은 차원이 768, 4096처럼 많음 — 아래는 이해용 2D 축소)

```text
                    긍정 감정
                        ↑
                  ★ 행복 ─ ★ 기쁨
                        │  (가까움)
                        │
    ────────────────────┼────────────────────→
                        │
              ● 고양이 ─● 강아지              ■ 자동차
                 (가까움)                      (멀리)
                        │
                        ↓
                      동물
```

**읽는 법**
- `●` `★` `■` = 단어 하나 = 의미 공간의 한 점
- 가까이 모인 점 = 의미가 비슷함 (동물끼리, 긍정 감정끼리)
- 멀리 떨어진 점 = 의미가 다름 (고양이 vs 자동차)
- 실제 모델은 2D가 아니라 수백~수천 개 좌표를 씀 — 원리만 “지도 위 좌표”와 같음


### 비유 2: 택배 상자 (Tokenization 노트 이어서)
-> [01. Tokenization - 택배 상자 비유](./01_tokenization.md#택배-상자-비유)

Tokenization 노트의 창고 비유를 그대로 이어감

```text
Tokenization:   "고양이" -> 4521번 선반
Embedding:      4521번 선반 -> 그 물건의 "성분표" (숫자 목록)
```

* 선반 번호(Token ID) = 이름만 있음
* 성분표(Embedding) = 그게 어떤 성격·의미를 가졌는지 숫자로 표현


## 전체 흐름 (LLM 안에서)

```text
"고양이가 귀엽다"
    ↓ Tokenization
[고양이] [가] [귀엽다]  →  [4521, 88, 1203]
    ↓ Embedding
[[0.2, -0.5, ...], [0.1, 0.3, ...], [0.4, 0.2, ...]]
    ↓ (다음 단계: Attention, Transformer ...)
문맥을 반영하며 의미를 더 정교하게 다듬음
```
> Embedding은 LLM이 "생각"을 시작하기 위한 첫 번째 번역 단계


## "벡터"?

벡터 = 숫자를 여러 개 나란히 적어 둔 것

### 예시 (실제는 훨씬 길다)
```text
고양이: [0.9, 0.1, 0.0]
강아지: [0.8, 0.2, 0.0]
자동차: [0.0, 0.1, 0.9]
```
    * 1번째 숫자 ≈ "동물일 가능성"
    * 3번째 숫자 ≈ "기계일 가능성"
-> 이건 사람이 일부러 정한 게 아니라, 모델이 대량의 글을 읽으며 스스로 학습한 패턴
∴ 비슷한 것끼리 숫자가 비슷해진다는 점


## Embedding이 하는 일 3가지
1. 번호 -> 의미
    Token ID만으로는 불가능한 "의미 표현"을 숫자로 만듦
2. 비슷함을 계산 가능하게
    "고양이와 강아지는 비슷하다"를 숫자 거리로 비교할 수 있음
3. 다음 단계의 재료
    Transformer·Attention은 이 숫자 묶음들을 입력으로 받아 문맥을 이해함


## 실생활에서 느껴지는 Embedding

    * 검색: "고양이 사료" 검색 시 "냥이 간식"도 잘 찾아줌 -> 비슷한 embedding끼리 가깝기 때문
    * 추천: "이 영화 좋아함" -> 비슷한 embedding의 다른 영화 추천
    * RAG: 문서를 embedding으로 저장해 두고, 질문과 가까운 문서를 찾음



## 내가 이해한 내용

Embedding은 token을 컴퓨터가 계산할 수 있는 의미 좌표로 바꾸는 것.

한 칸 한 칸의 의미보다 점들 사이의 관계를 알 수 있음
> 단어·token들이 모델 안에서 어떻게 묶이고, 얼마나 비슷하고, 어떤 방향으로 조합되는지 (유사도, 군집, 벡터연산)
> 각 숫자 하나가 가르키는 실체는 없고, 문맥 속에서 이 token이 다른 token과 어떻게 맞물리는지
